Unit tests
for AI prompts.

When you tweak a prompt, swap models, or upgrade an API version, how do you know your AI still behaves correctly? prompt-drift catches output regressions before your users do.

The problem nobody talks about

You test your code. You don't test your AI.

You tweak a prompt

A small wording change silently shifts the output format, tone, or structure.

You swap models

Moving from GPT-4 to Claude, or upgrading to a newer version, changes behavior in subtle ways.

You upgrade an API

Model providers quietly ship changes. Your carefully tuned outputs drift without warning.

You find out when a user complains.

How it works

Three commands. Works in any CI pipeline.

01

Record golden outputs

Run your prompts once. prompt-drift saves the outputs as your source of truth.

bash
$ prompt-drift record
Recording golden outputs (3 tests)
OK summarize_bullets -> .golden/summarize_bullets.json
OK sentiment_check -> .golden/sentiment_check.json
OK extract_entities -> .golden/extract_entities.json
Done. Commit your .golden/ directory.
02

Commit your golden files

The .golden/ directory is your locked expected behavior, version controlled alongside your code.

bash
$ git add .golden/ && git commit -m "lock golden outputs"
[main 3f2a1c] lock golden outputs
3 files changed, 87 insertions(+)
03

Check on every PR

prompt-drift re-runs your prompts and semantically scores them against the golden files. Drift fails the build.

bash
$ prompt-drift check
Checking prompts (3 tests)
PASS summarize_bullets score=0.91 threshold=0.80
FAIL sentiment_check score=0.42 threshold=0.80
Expected: positive
Actual: The overall sentiment appears quite positive...
PASS extract_entities score=0.88 threshold=0.80
1/3 tests failed.

Everything you need

No new infrastructure. No new accounts. Just a YAML file and your existing API key.

Semantic scoring

LLM-as-judge rates meaning similarity from 0.0 to 1.0. Not brittle string matching.

Per-test thresholds

Strict on one-word answers, lenient on open-ended summaries. Configurable per test.

File or inline

Reference prompt files, input files, or write everything inline. Mix and match freely.

Anthropic and OpenAI

Works with either provider. Switch anytime by changing one line in your config.

Golden files in git

Outputs are version controlled. Roll back AI behavior the same way you roll back code.

Mock provider

Try the full flow with zero API keys. No cost, no account, works offline.

Drop into CI
in one step

Works with GitHub Actions, GitLab CI, or any pipeline that runs shell commands. Add one step, set your secret, ship with confidence.

GitHub Actions GitLab CI CircleCI Any shell
.github/workflows/prompt-ci.yml
name: prompt regression
on: [push, pull_request]
jobs:
check:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v4
- run: pip install prompt-drift
- run: prompt-drift check
env:
ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

Get started in 60 seconds

1
pip install prompt-drift
2
prompt-drift init # edit prompt-ci.yaml
3
prompt-drift record # commit .golden/
4
prompt-drift check # run in CI on every PR
