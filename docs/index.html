The problem nobody talks about

You test your code. You don't test your AI.

✏️

You tweak a prompt

A small wording change silently shifts the output format, tone, or structure.

🔄

You swap models

Moving from GPT-4 to Claude — or to a newer version — changes behavior in subtle ways.

📦

You upgrade an API

Model providers quietly ship changes. Your carefully tuned outputs drift without warning.

😬 You find out when a user complains.

How it works

Three commands. Works in any CI pipeline.

01

Record golden outputs

Run your prompts once. prompt-drift saves the outputs as your source of truth.

$ prompt-drift record

Recording golden outputs (3 tests)

  OK summarize_bullets  -> .golden/summarize_bullets.json
     - Revenue grew 23% YoY
     - Margins expanded to 18%
     - Guidance raised for Q4

  OK sentiment_check    -> .golden/sentiment_check.json
     positive

  OK extract_entities   -> .golden/extract_entities.json
     ["John Smith", "Acme Corp", "New York"]

Done. Commit your .golden/ directory.
02

Commit your golden files

The .golden/ directory is your locked expected behavior — version controlled, just like your code.

$ git add .golden/ && git commit -m "lock golden outputs"
[main 3f2a1c] lock golden outputs
 3 files changed, 87 insertions(+)
03

Check on every PR

prompt-drift re-runs your prompts and semantically scores them against the golden files. Drift fails the build.

$ prompt-drift check

Checking prompts (3 tests)

  PASS summarize_bullets  score=0.91  threshold=0.80  (llm-judge)
  FAIL sentiment_check    score=0.42  threshold=0.80  (llm-judge)
       Expected: positive
       Actual:   The overall sentiment appears quite positive...
  PASS extract_entities   score=0.88  threshold=0.80  (llm-judge)

1/3 tests failed.

Simple config. Powerful control.

One YAML file defines your entire test suite.

prompt-ci.yaml 
provider: anthropic   # or openai
model: claude-haiku-4-5-20251001
threshold: 0.80        # global default
golden_dir: .golden

tests:
  - name: summarize_bullets
    prompt: "Summarize in 3 bullet points: {{input}}"
    input_file: inputs/article.txt
    threshold: 0.85

  - name: sentiment_check
    prompt: "Reply with one word: positive, negative, or neutral."
    input: "I love this product!"
    threshold: 0.95

  - name: customer_support
    prompt_file: prompts/support.txt
    input_file: inputs/complaint.txt
🎯

Per-test thresholds

Strict on single-word answers, lenient on open-ended summaries.

📁

File or inline

Reference prompt and input files, or write them inline. Mix and match.

🔀

Variable substitution

Use {{variable}} in prompts. {{input}} is automatically replaced.

🤖

LLM-as-judge scoring

Semantic similarity, not brittle string matching. Catches meaning drift.

Drop into CI in one step

Works with GitHub Actions, GitLab CI, or any pipeline that runs shell commands.

.github/workflows/prompt-ci.yml 
name: prompt regression

on: [push, pull_request]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install prompt-drift
      - run: prompt-drift check
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

2
